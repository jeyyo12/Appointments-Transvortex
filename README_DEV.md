# Dev Map (Where to edit what)

- Entry/bootstrap: `src/app/bootstrap.js` (wires Firebase init, routes, feature controllers)
- Tabs routing: `src/app/routes.js`
- Firebase init: `src/firebase/firebase-init.js`
- Auth/admin logic: `src/firebase/firebase-auth.js`
- Firestore helpers: `src/firebase/firestore.js`
- Appointments:
  - Controller/orchestration: `src/features/appointments/appointmentsController.js`
  - UI/render & delegation: `src/features/appointments/appointmentsView.js`
  - CRUD/service: `src/features/appointments/appointmentsService.js`
  - Model/status validation: `src/features/appointments/appointmentsModel.js`
- Pages feature: `src/features/pages/*`
- Invoice:
  - Click/download flow: `src/features/invoice/invoiceController.js`
  - Firestore invoice data: `src/features/invoice/invoiceService.js`
  - PDF rendering (jsPDF + Images/Invoice.png): `src/features/invoice/invoicePdf.js`
  - Template coordinates: `src/features/invoice/invoiceLayoutMap.js`
  - Preview (optional): `src/features/invoice/invoicePreview.js`
  - Data mapping: `src/features/invoice/invoiceMapper.js`
- UI helpers: `src/ui/*` (dom, notifications, modal, loading)
- Utils: `src/utils/*` (formatters, id generators, validation)
- Assets paths (GitHub Pages subfolder): use relative paths like `styles/invoice.css`, `Images/Invoice.png`
- Invoice button bug: check `appointmentsView.js` (data-apt-id) + `invoiceController.js`
- Firestore permissions: update rules in Firebase console; services payloads in `appointmentsService.js` / `invoiceService.js`
